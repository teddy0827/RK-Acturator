# ASML Scanner Overlay RK Parameter & Actuator 매핑 분석 작업기록

## 1. 프로젝트 배경

### 1.1 목적
ASML 스캐너 노광설비에서 Shot 성분 overlay misalign을 보정하기 위한 **20개 RK Parameter**와 이를 제어하는 **액추에이터(Scan, Lens, Scan Lens)** 간의 매핑 관계를 검증하고 시각화하는 것.

### 1.2 핵심 수식 (3차 다항식 Overlay 보정 모델)
```
dx = K1 + K3*x - K5*y + K7*x² + K9*x*y + K11*y² + K13*x³ + K15*x²*y + K17*x*y² + K19*y³
dy = K2 + K4*y + K6*x + K8*y² + K10*y*x + K12*x² + K14*y³ + K16*y²*x + K18*y*x² + K20*x³
```
- x = Slit 방향 (슬릿, 비스캔 방향)
- y = Scan 방향 (스캔 방향)

### 1.3 사용자의 초기 가설 (배경지식.xlsx)
좌표 의존성에 기반한 분류:
- **Wafer Stage**: 순수 y항 (K1,K5,K11,K19 / K2,K4,K8,K14)
- **Lens**: 순수 x항 (K3,K7,K13 / K6,K12,K20)
- **Scanning Lens**: 교차항 xy (K9,K15,K17 / K10,K16,K18)

## 2. 검증 과정

### 2.1 인터넷 검색 (1차)
SPIE, ResearchGate, ASML 공식 사이트 등에서 관련 논문/자료 검색.
- 검색 키워드: "ASML overlay RK parameter actuator", "intrafield overlay correction polynomial", "iHOPC" 등
- **결과**: 공개 자료에서는 RK 파라미터별 액추에이터 매핑을 명시한 자료를 찾기 어려웠음
- 일반적인 overlay 보정 구조(Translation, Rotation, Magnification, Trapezoid 등)는 확인 가능했으나 구체적 K-파라미터 명칭과 액추에이터 연결은 공개 문헌에 부족

### 2.2 초기 분석 결론 (수정 전)
좌표 기반 분류가 물리적으로 타당하다고 판단하여 **사용자 표가 정확하다고 결론** 내림.
- 순수 y항 → Wafer Stage (스캔방향은 스테이지가 제어)
- 순수 x항 → Lens (슬릿방향은 렌즈가 제어)
- 교차항 → Scan Lens (스캔 중 렌즈를 동적 변경)

> **이 결론은 이후 논문 검증에서 일부 틀렸음이 확인됨**

|---|-----------|-----------|
| K6 | Lens | **Scan** |

> **주의:** 논문 Table 1은 액추에이터 매핑 결과만 기재하고 있으며, 각 K 파라미터가 해당 액추에이터에 배정된 **물리적 이유는 설명하지 않음**. K6/K10/K16이 왜 Scan인지, K20이 왜 불가능한지에 대한 근거는 이 논문만으로는 확인 불가. 추후 ASML 내부 기술문서나 관련 후속 논문에서 확인 필요.

### 2.6 핵심 발견
- **dx (X방향)**: Scan 4개 / Lens 3개 / Scan Lens 3개 → 비교적 고르게 분배
- **dy (Y방향)**: Scan **7개** / Lens **1개(K12)** / Scan Lens **1개(K18)** / 없음 1개(K20)
- **결론**: 좌표 기반 분류(x항→Lens, y항→Scan, xy→Scan Lens)는 dx에서는 성립하지만, **dy에서는 성립하지 않음**
- dy 방향은 스캔 스테이지가 압도적으로 담당함
- 논문 p.10: "18 parameters (K1–K20, **excluding K17 and K20**)" — K17과 K20은 당시 field-to-field 보정에서 사용 불가

## 4. 각 K 파라미터 물리적 의미 상세

### 1차 (Linear)
| K | 의미 | 변형 형태 |
|---|------|---------|
| K1,K2 | Translation (평행이동) | 전체 Shot이 X 또는 Y로 동일하게 이동 |
| K3,K4 | Magnification (배율) | 중심에서 멀수록 벌어짐/좁아짐 |
| K5,K6 | Rotation (회전) | Shot 전체가 회전 |

### 2차 (Quadratic)
| K | 의미 | 변형 형태 |
|---|------|---------|
| K7,K8 | 2nd Order Magnification | x² 또는 y²에 비례 → **항상 같은 방향**으로 밀림 |
| K9,K10 | Trapezoid (사다리꼴) | xy에 비례 → 위/아래 반대 방향 벌어짐 |
| K11,K12 | Bow (활 모양) | y² 또는 x²에 비례 → 가장자리가 같은 방향으로 밀림 |

### 3차 (Cubic)
| K | 의미 | 변형 형태 |
|---|------|---------|
| K13,K14 | 3rd Order Magnification | x³ 또는 y³ → S자형 왜곡 |
| K15,K16 | Accordion (아코디언) | x²y 또는 y²x → 주름 형태 |
| K17,K18 | C-shape Distortion | xy² 또는 yx² → C자 형태 |
| K19,K20 | 3rd Order Flow | y³ 또는 x³ → S자 흐름 |

## 5. 액추에이터 배정의 물리적 근거

### 5.1 핵심 원리: 슬릿 스캔 노광의 물리적 구조

스캐너에서는 좁은 슬릿(약 26mm × 6mm)을 통해 노광한다. 웨이퍼가 y방향(스캔방향)으로 이동하면서 슬릿을 통과하며, 이 구조가 액추에이터 배정을 물리적으로 결정한다.

| 액추에이터 | 물리적 메커니즘 | 보정 가능한 항 | 이유 |
|---|---|---|---|
| **Lens** | 렌즈 엘리먼트의 위치/기울기 조정 (스캔 중 **고정**) | 순수 x항 (x, x², x³) | 슬릿 폭 전체가 동시에 노광되므로, x방향 왜곡은 렌즈 형상으로만 보정 가능 |
| **Scan (Stage)** | 웨이퍼/레티클 스테이지의 위치·속도를 스캔 중 **동적으로** 변조 | 순수 y항 (y, y², y³) | 스캔 진행에 따라 스테이지 위치를 실시간 조절하면 y방향 보정 가능 |
| **Scan Lens** | 렌즈 엘리먼트를 스캔 위치(y)에 **동기화하여** 동적으로 변경 | 교차항 (xy, x²y, xy²) | x방향 렌즈 보정이 y위치에 따라 달라져야 하므로, 스캔 중 렌즈를 변경해야 함 |

### 5.2 근거 출처

**① Nikon D-MAC 문서 (가장 명시적인 공개 자료)**
- K9(trapezoid), K17(C-shape): "across the scan slit" (슬릿 가로방향) — 렌즈 엘리먼트를 노광 중에 상하 이동 및 틸팅하여 보정
- K18(C-shape): "along the scan slit" (스캔 방향) — 더 최근에 추가된 기능으로, 스캔 동기화 렌즈 조정 필요
- 출처: Nikon NSR-S322F 제품 페이지, Nikon eReview Spring 2012

**② IBM/IMEC SPIE 논문 (2019)**
> "Intrafield overlay corrections are implemented by reticle and/or wafer stage modulations during the exposure scan."
- 스테이지 변조가 인트라필드 보정의 주요 메커니즘임을 직접 확인
- 출처: Pollak et al., "Characterization of EUV image fading induced by overlay corrections" (2019)

**③ van den Brink et al., Applied Optics (1996) — 슬릿 평균(Slit Averaging) 효과**
- y방향(스캔방향) 렌즈 왜곡은 슬릿을 통과하면서 적분/평균화됨
- x방향(슬릿방향) 왜곡은 평균화되지 않고 그대로 이미지에 반영됨
- → 렌즈로는 y방향 보정이 사실상 불가능 (슬릿 적분으로 상쇄됨)
- 출처: "Effect of lens distortion in optical step-and-scan lithography," Applied Optics 35(4), 690 (1996)

**④ Macronix 특허 (US20060238761A1)**
- Asymmetric magnification/rotation은 "레티클 스테이지와 웨이퍼 스테이지의 상대적 운동"이 원인
- → 스테이지 동기화 파라미터임

### 5.3 불일치 4개 항목의 물리적 이유

**K6 (dy = K6·x, Rotation Y) → Scan (Lens 아님)**
- 비대칭 회전(asymmetric rotation)에 해당하며, 특허에서 "레티클 스테이지와 웨이퍼 스테이지의 상대적 운동"으로 발생한다고 기재
- 물리적으로 스테이지의 미세한 yaw(요 회전)으로 보정 가능 → 렌즈 조정 불필요
- 슬릿 적분 후, x에 대해 일정한 dy 성분은 스테이지 회전으로 환원됨

**K10 (dy = K10·y·x, Trapezoid Y) → Scan (Scan Lens 아님)**
- 교차항(yx)이지만 dy 방향이므로, 슬릿 적분 후 y 의존적 스테이지 보정으로 환원됨
- 슬릿 폭에 걸친 x 의존성이 평균화되어 순수 y 의존 보정이 됨
- Canon FPD 문서에서 확인: "스테이지 스캔 속도를 y 위치의 함수로 변조" 가능

**K16 (dy = K16·y²·x, Accordion Y) → Scan (Scan Lens 아님)**
- K10과 같은 원리: dy 방향의 y²x 항은 슬릿 적분 후 y² 의존적 스테이지 속도 변조로 보정 가능

**K20 (dy = K20·x³, 3rd Order Flow Y) → 없음**
- 순수 x³항인데 dy 방향 → 렌즈로 보정하려면 y방향 변위를 x³에 비례하게 만들어야 하나, 렌즈 엘리먼트는 dx(슬릿방향 변위)는 제어 가능하지만 dy(스캔방향 변위)를 x 의존적으로 만드는 메커니즘이 없음
- 스테이지는 y방향을 제어하지만 x에 의존한 보정은 불가
- 2013년 당시 어떤 액추에이터로도 보정 불가능 → "None"

### 5.4 핵심 통찰: 슬릿 적분이 차원을 바꾼다

> **오버레이 다항식에서 xy 교차항으로 보이는 것이, 슬릿 적분 후에는 순수 y 의존적 스테이지 보정으로 환원될 수 있다.**

이것이 K10, K16이 교차항임에도 Scan Lens가 아닌 Scan으로 배정된 이유이다. dy 방향에서는 슬릿 적분에 의해 x 의존성이 평균화되므로, 스테이지 속도/위치 변조만으로 충분하다.

반면 dx 방향의 교차항(K9, K15, K17)은 x방향 변위를 y에 따라 달리해야 하므로, 렌즈를 스캔 중에 동적으로 바꿔야만 한다 → Scan Lens.

**정리:**
- **dx 교차항** → Scan Lens 필요 (x방향 변위가 y에 따라 변해야 하므로 동적 렌즈 조정 필수)
- **dy 교차항** → Scan으로 충분 (슬릿 적분으로 x 의존성이 평균화되어 스테이지 변조로 환원)
- **dy 순수 x항 (K6)** → Scan (스테이지 회전/yaw로 환원)
- **dy 순수 x³항 (K20)** → 보정 불가 (어떤 액추에이터로도 dy의 순수 고차 x 의존성 생성 불가)

## 6. 향후 고도화 방향

### 6.1 데이터 관련
- 실제 스캐너 overlay 측정 데이터를 입력하여 20개 K 파라미터를 Least Squares로 피팅하는 기능
- CPE(Correction Per Exposure) 적용 전/후 비교 시각화
- 웨이퍼 전체 필드 맵에서 각 필드별 K 파라미터 분포 히트맵

### 6.2 시각화 관련
- 인터랙티브 슬라이더로 각 K 값을 실시간 조절하며 격자 변형 확인 (HTML+JS 또는 Plotly)
- 여러 K를 동시에 적용한 복합 변형 시각화
- 벡터 필드(화살표 맵) 모드 추가/토글

### 6.3 분석 관련
- 20개 파라미터 중 어떤 것이 overlay 기여도가 큰지 sensitivity 분석
- 액추에이터 range 한계를 반영한 보정 가능/불가능 영역 표시
- 최신 스캐너(NXT:2100i 등)에서 K20 등 추가 액추에이터 지원 여부 조사
- Reticle heating에 의한 K 파라미터 시간 변화 모델링

### 6.4 확인이 필요한 사항
- 본 논문은 2013년 기준이므로 최신 스캐너에서 액추에이터 매핑이 변경되었을 가능성 있음
- K20이 현재 최신 세대에서는 보정 가능해졌을 수 있음 (NXT:2100i의 new lens distortion manipulator)
- K17은 논문 p.10에서 "excluding K17 and K20"으로 당시 field-to-field에서 제외되었으나, Table 1에서는 Scan Lens로 기재 → lot average iHOPC에서는 사용 가능하지만 per-field 동적 보정에서는 불가능했을 수 있음
